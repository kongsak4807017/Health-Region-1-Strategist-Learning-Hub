# Root Cause Analysis for Public Health

## นิยามและหลักการ

**Root Cause Analysis (RCA)** คือ กระบวนการวิเคราะห์เชิงระบบเพื่อค้นหาสาเหตุที่แท้จริง (root causes) ของปัญหาสาธารณสุข โดยไม่ใช่เพียงแค่อาการที่ปรากฏ (symptoms)

**หลักการสำคัญ:**
1. มุ่งเน้นที่ระบบ ไม่ใช่บุคคล (System focus, not blame)
2. ค้นหาสาเหตุเชิงลึก ไม่ใช่ผิวเผิน
3. ใช้หลักฐานข้อมูลประกอบการวิเคราะห์
4. พิจารณาหลายมิติ ( biological, behavioral, social, environmental, policy)

## กรอบแนวคิดสำหรับ RCA ด้านสาธารณสุข

### 1. Social-Ecological Model (SEM)

```
┌─────────────────────────────────────┐
│  Public Policy (นโยบายสาธารณะ)      │
│  กฎหมาย การจัดสรรงบ ระบบบริการ      │
├─────────────────────────────────────┤
│  Community (ชุมชน)                  │
│  ความสัมพันธ์ นorms สภาพแวดล้อม     │
├─────────────────────────────────────┤
│  Institutional (สถาบัน)             │
│  กฎระเบียบ วัฒนธรรมองค์กร          │
├─────────────────────────────────────┤
│  Interpersonal (ระหว่างบุคคล)        │
│  ครอบครัว เพื่อน คู่สมรส            │
├─────────────────────────────────────┤
│  Individual (บุคคล)                  │
│  ความรู้ ทัศนคติ พฤติกรรม           │
└─────────────────────────────────────┘
```

**การใช้ SEM ใน RCA:**
- วิเคราะห์ปัจจัยเสี่ยงในแต่ละระดับ
- ระบุจุดที่สามารถแทรกแซงได้
- พิจารณาปฏิสัมพันธ์ระหว่างระดับ

### 2. Commission on Social Determinants of Health (CSDH) Framework

**กลไกที่สร้างความไม่เสมอภาค:**

| Level | Factors | Examples in Thailand |
|-------|---------|---------------------|
| Structural | การกีดกัน การแบ่งชนชั้น | รายได้ การศึกษา ชนบท-เมือง |
| Intermediary | วัสดุ, สังคม, จิตวิทยา | ที่อยู่อาศัย การเข้าถึงอาหาร |
| Health System | การเข้าถึงบริการ | รพ.สต. สิทธิ UCS |

### 3. 5 Whys Technique

เทคนิคถาม "ทำไม" ซ้ำ 5 ครั้งเพื่อลงลึกถึงสาเหตุ

**ตัวอย่าง: ปัญหาอัตราตายมารดาสูงในจังหวัด X**

```
ปัญหา: อัตราตายมารดาสูง (80 ต่อ 100,000 เกิดมีชีพ)

ทำไม? → มีภาวะแทรกซ้อนระหว่างตั้งครรภ์และคลอดมาก
         
ทำไม? → มารดาไม่ได้รับการดูแลก่อนคลอดที่เหมาะสม
         
ทำไม? → มารดาบางกลุ่มไม่เข้ารับบริการ ANC (ฝากครรภ์)
         
ทำไม? → ไม่มีค่าเดินทางและขาดความรู้เรื่องสิทธิ UCS
         
ทำไม? → ระบบข้อมูลไม่สามารถระบุกลุ่มเสี่ยงได้ล่วงหน้า
         
Root Cause: ขาดระบบ early warning สำหรับมารดากลุ่มเสี่ยง
```

### 4. Fishbone Diagram (Ishikawa)

วิเคราะห์สาเหตุตามหมวดหมู่

```
                           Effect
                    (ปัญหาที่ศึกษา)
                              |
                              v
    People ----------+--------+-------- Process
    (คน)             /        \          (กระบวนการ)
                    /          \
    Place ----------+------------+-------- Policy
    (สถานที่)                                     (นโยบาย)
```

**Categories for Public Health:**

**People (คน):**
- ประชากรกลุ่มเป้าหมาย
- ผู้ให้บริการสุขภาพ
- ผู้กำหนดนโยบาย

**Process (กระบวนการ):**
- กระบวนการบริการ
- การส่งต่อผู้ป่วย
- การจัดการข้อมูล

**Policy (นโยบาย):**
- นโยบายระดับชาติ
- ระเบียบปฏิบัติ
- การจัดสรรทรัพยากร

**Place (สถานที่/บริบท):**
- โครงสร้างพื้นฐาน
- ภูมิศาสตร์
- สภาพแวดล้อมทางสังคม

### 5. Pareto Analysis (80/20 Rule)

**หลักการ:** 20% ของสาเหตุสร้าง 80% ของปัญหา

**ขั้นตอน:**
1. รวบรวมข้อมูลความถี่ของแต่ละสาเหตุ
2. เรียงลำดับจากมากไปน้อย
3. คำนวณ cumulative percentage
4. ระบุ "vital few" (20%) ที่สร้างปัญหาหลัก

**ตัวอย่าง: สาเหตุการตายในโรงพยาบาลชุมชน**

| Cause | Frequency | Cumulative % |
|-------|-----------|--------------|
| ติดเชื้อในกระแสเลือด | 40 | 40% |
| หัวใจล้มเหลว | 30 | 70% |
| หลอดเลือดสมอง | 15 | 85% |
| อื่นๆ | 15 | 100% |

→ Focus on sepsis and heart failure (70% of deaths)

### 6. Fault Tree Analysis (FTA)

วิเคราะห์เหตุการณ์ที่ไม่พึงประสงค์แบบลอจิก

```
                  [Undesired Event]
                         |
           +-------------+-------------+
           |                           |
      [AND Gate]                  [OR Gate]
           |                           |
    +------+------+             +------+------+
    |             |             |             |
[Event A]    [Event B]     [Event C]     [Event D]
(ต้องเกิดทั้งคู่)           (เกิดอย่างใดอย่างหนึ่ง)
```

### 7. Barrier Analysis

วิเคราะห์อุปสรรคที่ป้องกันไม่ให้บรรลุเป้าหมาย

**ประเภทของ Barriers:**
- **Physical**: ระยะทาง อุปกรณ์
- **Behavioral**: ความรู้ ทัศนคติ
- **Administrative**: ระเบียบ ขั้นตอน
- **Economic**: ค่าใช้จ่าย การจัดสรรงบ

**ตัวอย่าง: Barriers ต่อการเข้าถึงบริการสุขภาพ**

```
[เป้าหมาย: เข้าถึงบริการ]
         |
    [Barrier 1] ระยะทาง
         |→ แก้ไข: รพ.สต. รถพยาบาล
         |
    [Barrier 2] ค่าใช้จ่าย
         |→ แก้ไข: UCS ฟรี
         |
    [Barrier 3] ความรู้
         |→ แก้ไข: ให้ข้อมูล
         |
    [Barrier 4] ทัศนคติ
              → แก้ไข: รณรงค์
```

## กระบวนการทำ RCA อย่างเป็นระบบ

### Phase 1: Define the Problem

**SMART Problem Definition:**
- **S**pecific: ชัดเจนเฉพาะเจาะจง
- **M**easurable: วัดได้
- **A**chievable: แก้ไขได้
- **R**elevant: สำคัญต่อเป้าหมาย
- **T**ime-bound: มีกรอบเวลา

**ตัวอย่างที่ดี:**
"อัตราการตายจากโรคหัวใจในอำเภอ X เพิ่มจาก 50 เป็น 80 ต่อ 100,000 ประชากร ในช่วงปี 2564-2566 สูงกว่าค่าเฉลี่ยจังหวัด 2 เท่า"

### Phase 2: Data Collection

**แหล่งข้อมูล:**
1. ข้อมูลเชิงปริมาณ (Quantitative)
   - ข้อมูลทะเบียน (mortality, morbidity)
   - ข้อมูลบริการ (utilization rates)
   - ข้อมูลทรัพยากร (workforce, facilities)

2. ข้อมูลเชิงคุณภาพ (Qualitative)
   - การสัมภาษณ์ผู้มีส่วนได้ส่วนเสีย
   - Focus group discussions
   - การสังเกตการณ์
   - การศึกษาเอกสาร

### Phase 3: Identify Causal Factors

**เทคนิคการระบุสาเหตุ:**
1. Brainstorming
2. Affinity diagram
3. Timeline analysis
4. Change analysis (อะไรเปลี่ยนไปก่อนเกิดปัญหา)

### Phase 4: Determine Root Causes

**เกณฑ์ของ Root Cause:**
- ถ้าแก้ไขได้ ปัญหาจะไม่กลับมา (or significantly reduced)
- เป็นสาเหตุลึกสุดที่สามารถควบคุมได้
- มีหลักฐานสนับสนุน

### Phase 5: Develop Recommendations

**SMART Recommendations:**
- ระบุผู้รับผิดชอบ
- กำหนด Timeline
- ตั้งตัวชี้วัดความสำเร็จ
- ประเมินทรัพยากรที่ต้องใช้

### Phase 6: Implement and Monitor

- สร้าง action plan
- ติดตามความคืบหน้า
- ประเมินผลลัพธ์
- ปรับปรุงตามความเหมาะสม

## เครื่องมือเสริม

### Causal Loop Diagram (CLD)

แสดงความสัมพันธ์เชิงเหตุผลและ feedback loops

```
    [Poverty] ───────(+──────> [Malnutrition]
        ^                         │
        │                         (+)
        │                         v
        └──────────(+)──────── [Disease]
```

(+): ทิศทางเดียวกัน
(-): ทิศทางตรงกันข้าม

### The 5 Whys + Evidence Matrix

รวม 5 Whys กับหลักฐานข้อมูล

| Why Level | Cause | Evidence Source | Data Quality |
|-----------|-------|-----------------|--------------|
| 1 | ตายจากโรคปอด | Death certificate | High |
| 2 | ปอดอักเสบเรื้อรัง | Medical records | Medium |
| 3 | สูบบุหรี่ | Survey data | High |
| 4 | ไม่เข้าถึงบริการเลิกบุหรี่ | Service data | Medium |
| 5 | ไม่มีโปรแกรมในพื้นที่ | Policy review | High |

## ข้อควรระวังใน RCA

1. **Confirmation bias**: มองหาข้อมูลที่ยืนยันความเชื่อเดิม
2. **Oversimplification**: ลดความซับซ้อนของปัญหามากเกินไป
3. **Blame culture**: โทษบุคคลแทนที่จะดูระบบ
4. **Insufficient data**: วิเคราะห์โดยไม่มีข้อมูลเพียงพอ
5. **Analysis paralysis**: วิเคราะห์มากเกินไปจนไม่ลงมือทำ

## ตัวอย่างการใช้ RCA ในประเด็นสาธารณสุขไทย

### ตัวอย่าง: ปัญหาเด็กอ้วนในประเทศไทย

```
ปัญหา: อัตราเด็กอ้วนเพิ่มขึ้น (จาก 8% เป็น 14% ใน 10 ปี)

Level 1 (Immediate): กินมาก + ออกกำลังกายน้อย
Level 2 (Underlying): อาหารขยะ น้ำตาล เครื่องดื่มหวาน
Level 3 (Structural): การโฆษณา ราคาอาหาร สภาพแวดล้อม
Level 4 (Policy): กฎหมายอาหาร นโยบายโรงเรียน การศึกษา

Root Causes:
1. ขาดกฎหมายควบคุมการโฆษณาอาหารเด็ก
2. โรงเรียนขายอาหารไม่มีประโยชน์
3. สภาพแวดล้อมไม่เอื้อต่อการออกกำลังกาย
```
