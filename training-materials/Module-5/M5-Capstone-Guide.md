## ขั้นตอนที่ 1: Problem Definition

### 1.1 การเลือกปัญหา

**หลักการเลือก:**
- มีข้อมูลพอเพียง (หรือสามารถหาได้)
- มีความสำคัญต่อสาธารณสุข
- มีโอกาสแก้ไขได้
- น่าสนใจสำหรับทีม

**คำถามที่ต้องตอบ:**
```
1. ปัญหานี้ส่งผลกระทบต่อใคร? กี่คน?
2. มีข้อมูลอะไรบ้างที่รองรับว่าเป็นปัญหา?
3. ทำไมปัญหานี้ถึงสำคัญ?
4. มีแนวโน้มว่าจะแย่ลงหรือดีขึ้น?
```

### 1.2 การเขียน Problem Statement

**โครงสร้าง:**
```
ในขณะนี้ [กลุ่มเป้าหมาย] ในพื้นที่ [ระบุพื้นที่] 
กำลังประสบกับ [ปัญหา/สถานการณ์] 
ซึ่งมี [ผลกระทบ/ความรุนแรง] 
เนื่องจาก [สาเหตุเบื้องต้น/ปัจจัยสำคัญ]
```

**ตัวอย่างที่ดี:**
> ในขณะนี้ หญิงตั้งครรภ์ในพื้นที่ห่างไกลของจังหวัดแม่ฮ่องสอน (ประมาณ 500 คนต่อปี) มีอัตราการตายระหว่างตั้งครรภ์และการคลอดสูงถึง 180 ต่อ 100,000 การคลอด ซึ่งสูงกว่าค่าเฉลี่ยประเทศ 3 เท่า ส่งผลให้มีแม่เสียชีวิต 9 คนต่อปี เนื่องจากการเข้าถึงบริการฝากครรภ์คุณภาพไม่เพียงพอ และระบบการส่งต่อไม่มีประสิทธิภาพ

**ตัวอย่างที่ไม่ดี:**
> อัตราตายมารดาในจังหวัดแม่ฮ่องสอนสูงมาก ต้องแก้ไขด่วน

## ขั้นตอนที่ 2: Data Collection & Analysis

### 2.1 การรวบรวมข้อมูล

**แหล่งข้อมูลหลักที่ควรใช้:**

| ประเภท | แหล่งข้อมูล | ตัวอย่างข้อมูล |
|--------|-------------|---------------|
| ข้อมูลมรรตัว | MOPH HDC | อัตราตาย อัตราป่วย |
| ข้อมูลบริการ | NHSO | ค่าใช้จ่าย การเข้าถึง |
| ข้อมูลประชากร | NSO | ขนาดกลุ่มเป้าหมาย |
| ข้อมูลบริบท | สำรวจ | พฤติกรรม ความรู้ |

### 2.2 การตรวจสอบคุณภาพข้อมูล (TRUST Framework)

**T - Timely:**
- ข้อมูลเป็นปัจจุบันหรือไม่?
- มีการ update ล่าสุดเมื่อไหร่?

**R - Reliable:**
- แหล่งที่มาน่าเชื่อถือหรือไม่?
- วิธีเก็บข้อมูลมีมาตรฐานหรือไม่?

**U - Useful:**
- ข้อมูลตอบคำถามที่ตั้งไว้หรือไม่?
- มีความละเอียดพอหรือไม่?

**S - Specific:**
- ระบุกลุ่มเป้าหมายชัดเจนหรือไม่?
- มี breakdown ตามความจำเป็น?

**T - Triangulated:**
- มีข้อมูลจากหลายแหล่งไหม?
- ข้อมูลสอดคล้องกันหรือไม่?

### 2.3 การวิเคราะห์ช่องโหว่ข้อมูล

**ตัวอย่างการวิเคราะห์:**

| ข้อมูลที่ต้องการ | มี/ไม่มี | แหล่งที่มี | คุณภาพ | ช่องโหว่ |
|-----------------|---------|-----------|--------|---------|
| อัตราตายมารดา | มี | สสจ. | ดี | ล่าช้า 1 ปี |
| การฝากครรภ์ | มี | HDC | ปานกลาง | ไม่สมบูรณ์ |
| ความรู้หญิงตั้งครรภ์ | ไม่มี | - | - | ต้องสำรวจ |

## ขั้นตอนที่ 3: Root Cause Analysis

### 3.1 การใช้เครื่องมือ RCA

**ควรใช้หลายเครื่องมือร่วมกัน:**

#### 5 Whys
ใช้สำหรับ: วิเคราะห์ลึกถึง root cause ของปัญหาเฉพาะ

**ตัวอย่าง:**
```
ปัญหา: หญิงตั้งครรภ์ไม่มาฝากครรภ์

ทำไม? → ไม่เห็นความสำคัญ
    ↓
ทำไม? → ไม่เข้าใจอันตรายของการไม่ฝาก
    ↓
ทำไม? → ไม่ได้รับข้อมูลจาก รพ.สต.
    ↓
ทำไม? → อสม. ไม่มีเวลาเยี่ยมบ้าน
    ↓
ทำไม? → อสม. ดูแลพื้นที่กว้างเกินไป

Root Cause: สัดส่วน อสม.ต่อหลังคาเรือนไม่เพียงพอ
```

#### Fishbone Diagram
ใช้สำหรับ: วิเคราะห์สาเหตุในหลายมิติ

```
                    [ผู้ป่วย DM ควบคุมไม่ได้]
                             |
    +------------+-----------+-----------+------------+
    |            |           |           |            |
  People       Process      Policy       Place      Technology

People: ผู้ป่วย, ผู้ดูแล, บุคลากร
Process: การตรวจ, การรักษา, การ follow-up
Policy: ยา, บุคลากร, งบประมาณ
Place: บ้าน, ชุมชน, สถานบริการ
Technology: เครื่องมือ, ระบบ IT
```

#### Causal Loop Diagram
ใช้สำหรับ: แสดงความสัมพันธ์เชิงระบบและ feedback loops

```
┌─────────────┐       +       ┌─────────────┐
│  ผู้ป่วย    │──────────────→│  ภาวะแทรก   │
│  DM มาก    │               │  ซ้อนเพิ่ม  │
└──────┬──────┘               └──────┬──────┘
       │                            │
       │ +                          │ +
       ▼                            ▼
┌─────────────┐               ┌─────────────┐
│   ค่าใช้    │               │   ภาระ      │
│  จ่ายสูง   │←──────────────│  บุคลากร    │
└─────────────┘       -       └─────────────┘
       │
       │ -
       ▼
┌─────────────┐
│  การดูแล    │
│  เชิงรุก    │
└─────────────┘
```

## ขั้นตอนที่ 4: Policy Analysis

### 4.1 Policy Triangle Analysis

**Content (เนื้อหา):**
- ปัญหาที่ต้องการแก้ไขคืออะไร?
- เป้าหมายของนโยบายคืออะไร?
- กลยุทธ์หลักคืออะไร?

**Context (บริบท):**
- การเมือง: รัฐบาล/ฝ่ายค้าน สนับสนุนหรือไม่?
- เศรษฐกิจ: มีงบประมาณพอหรือไม่?
- สังคม: ประชาชนเห็นด้วยหรือไม่?
- เทคโนโลยี: มีเทคโนโลยีรองรับหรือไม่?

**Process (กระบวนการ):**
- ใครเป็นผู้กำหนดวาระ?
- การตัดสินใจเป็นอย่างไร?
- การดำเนินงานมีประสิทธิภาพหรือไม่?

**Actors (ผู้มีส่วนได้ส่วนเสีย):**
- ใครได้ประโยชน์? ใครเสียประโยชน์?
- ใครมีอำนาจตัดสินใจ?
- ใครมีอิทธิพลต่อการดำเนินงาน?

## ขั้นตอนที่ 6: Writing & Presentation

### 6.1 โครงสร้างรายงาน

```
หน้าปก
สารบัญ
บทที่ 1: บทนำ (2-3 หน้า)
บทที่ 2: การวิเคราะห์สถานการณ์ (5-7 หน้า)
บทที่ 3: การวิเคราะห์สาเหตุ (5-7 หน้า)
บทที่ 4: การวิเคราะห์นโยบาย (4-5 หน้า)
บทที่ 5: ข้อเสนอแนะ (6-8 หน้า)
บทที่ 6: สรุป (1-2 หน้า)
ภาคผนวก
References
```

### 6.2 การเขียน Policy Brief

**โครงสร้าง 1 หน้า:**
```
[หัวเรื่อง]

บริบท: 2-3 ประโยค

ปัญหา: Problem Statement

ข้อเสนอแนะ:
1. ...
2. ...
3. ...

ผลประโยชน์: ...

ติดต่อ: ...
```

### 6.3 การนำเสนอ

**โครงสร้างสไลด์ (10 สไลด์):**

1. Title (ปัญหา ทีม)
2. Agenda
3. Problem Statement + ข้อมูลสำคัญ
4. Root Cause (ภาพรวม)
5. Policy Analysis (สรุป)
6. Recommended Policy
7. Theory of Change
8. Implementation Plan
9. Expected Outcomes
10. Q&A

**เคล็ดลับการนำเสนอ:**
- ใช้รูปภาพมากกว่าตัวอักษร
- ตัวเลขสำคัญใส่ตัวหนา
- ซ้อมอย่างน้อย 1 ครั้ง
- เตรียมตอบคำถามยากๆ
