## 1. ความแตกต่างระหว่าง Symptom กับ Root Cause

### 1.1 นิยาม

| | Symptom (อาการ) | Root Cause (สาเหตุเชิงลึก) |
|---|----------------|--------------------------|
| **นิยาม** | สิ่งที่มองเห็น/วัดได้ | สาเหตุที่แท้จริงของปัญหา |
| **ลักษณะ** | ผลลัพธ์ (Outcome) | ปัจจัยที่ก่อให้เกิด |
| **การแก้ไข** | บรรเทาอาการ | แก้ไขที่ต้นตอ |
| **ความยั่งยืน** | อาจกลับมาเป็นอีก | แก้แล้วไม่กลับมา |

### 1.2 ตัวอย่างในบริบทสาธารณสุข

**ตัวอย่างที่ 1: อัตราตายมารดาสูง**

```
Symptom (อาการที่เห็น)
↓
อัตราตายมารดาสูง 80/100,000

Immediate Cause (สาเหตุทันที)
↓
มีภาวะแทรกซ้อนระหว่างตั้งครรภ์ (เลือดออก eclampsia)

Underlying Cause (สาเหตุเบื้องต้น)
↓
มารดาไม่ได้รับการดูแลก่อนคลอดที่เหมาะสม

Root Cause (สาเหตุเชิงลึก)
↓
ขาดระบบ early warning สำหรับมารดากลุ่มเสี่ยง
```

**ตัวอย่างที่ 2: โรคเบาหวานไม่ควบคุม**

```
Symptom:
HbA1c > 9% ใน 40% ของผู้ป่วย

Immediate Cause:
ผู้ป่วยไม่ทานยาตามที่แพทย์สั่ง

Underlying Cause:
ไม่เข้าใจโรค ขาดการติดตามผล

Root Cause:
ระบบการศึกษาผู้ป่วยและ case management ไม่มีประสิทธิภาพ
```

### 1.3 ทำไมต้องหา Root Cause?

**ปัญหาของการแก้ที่ Symptom:**

| การแก้ที่อาการ | ผลลัพธ์ |
|--------------|---------|
| เพิ่มงบซื้อเครื่องมือแพทย์ | ใช้ไม่คุ้มค่า ถ้าไม่มีคนรักษา |
| ตักเตือนผู้ป่วย | ไม่ยั่งยืน กลับมาเป็นเหมือนเดิม |
| เพิ่มโรงพยาบาล | ไม่แก้ปัญหา geographic access |

**ประโยชน์ของการแก้ที่ Root Cause:**
- ✅ แก้ครั้งเดียว ผลยั่งยืน
- ✅ ประหยัดทรัพยากรในระยะยาว
- ✅ ป้องกันปัญหาในอนาคต
- ✅ สร้างระบบที่ robust

## 2. ระดับของสาเหตุ (Levels of Causes)

### 2.1 โมเดล 3 ระดับ

```
┌─────────────────────────────────────┐
│  ROOT CAUSES (สาเหตุเชิงลึก)         │
│  • นโยบาย                            │
│  • โครงสร้างระบบ                     │
│  • วัฒนธรรมองค์กร                    │
│  • การจัดสรรทรัพยากร                 │
├─────────────────────────────────────┤
│  UNDERLYING CAUSES (สาเหตุเบื้องต้น) │
│  • กระบวนการทำงาน                    │
│  • ทักษะบุคลากร                       │
│  • การสื่อสาร                         │
│  • สภาพแวดล้อม                       │
├─────────────────────────────────────┤
│  IMMEDIATE CAUSES (สาเหตุทันที)      │
│  • พฤติกรรมบุคคล                      │
│  • การใช้อุปกรณ์                      │
│  • การตัดสินใจในเวลานั้น             │
│  • สภาพร่างกาย                       │
└─────────────────────────────────────┘
```

### 2.2 ตัวอย่างการวิเคราะห์ระดับต่างๆ

**ปัญหา: การติดเชื้อในโรงพยาบาลสูง**

| ระดับ | สาเหตุ | ตัวอย่าง |
|------|-------|---------|
| Immediate | พฤติกรรม | พยาบาลลืมล้างมือ |
| Underlying | กระบวนการ | ไม่มี protocol ที่ชัดเจน |
| Root | ระบบ | ขาดวัฒนธรรมความปลอดภัย |

**ปัญหา: เด็กอ้วนเพิ่มขึ้น**

| ระดับ | สาเหตุ | ตัวอย่าง |
|------|-------|---------|
| Immediate | พฤติกรรม | กินจังก์ฟู้ด ไม่ออกกำลังกาย |
| Underlying | สภาพแวดล้อม | โรงเรียนขายอาหารขยะ |
| Root | นโยบาย | ไม่มีกฎหมายควบคุมการโฆษณาอาหาร |

### 2.3 การเลือกระดับสำหรับการแก้ไข

**เกณฑ์การเลือก:**

| เกณฑ์ | Immediate | Underlying | Root |
|------|-----------|------------|------|
| ความเร่งด่วน | ✅ สูง | ⚠️ ปานกลาง | ❌ ต่ำ |
| ผลยั่งยืน | ❌ ต่ำ | ⚠️ ปานกลาง | ✅ สูง |
| ต้นทุน | ✅ ต่ำ | ⚠️ ปานกลาง | ❌ สูง |
| เวลาที่ใช้ | ✅ น้อย | ⚠️ ปานกลาง | ❌ มาก |
| ความยาก | ✅ ง่าย | ⚠️ ปานกลาง | ❌ ยาก |

**กลยุทธ์ที่ดีที่สุด:** แก้ทุกระดับแบบบูรณาการ

## 3. Systems Thinking

### 3.1 นิยาม

Systems Thinking คือ การมองปัญหาเป็นระบบที่ส่วนต่างๆ มีความสัมพันธ์เชื่อมโยงกัน ไม่ใช่เหตุการณ์แยกเดี่ยว

### 3.2 หลักการสำคัญ

#### A. มองภาพรวม (Big Picture)
```
❌ ผิด: "แพทย์ขาด ก็แค่สรรหาแพทย์เพิ่ม"

✅ ถูก: "แพทย์ขาด เกิดจากอะไร?"
   → แพทย์ย้ายไปเอกชน (ต้นทุนชีวิตสูง)
   → การศึกษาไม่ตอบโจทย์พื้นที่
   → ระบบ incentives ไม่ดี
```

#### B. เห็นความสัมพันธ์ (Interconnections)
```
┌─────────────────────────────────────────┐
│                                         │
│   ความยากจน ←──────→ สุขภาพไม่ดี       │
│       ↑                   ↑             │
│       │                   │             │
│       └────→ การศึกษาต่ำ ←┘             │
│                                         │
└─────────────────────────────────────────┘
```

#### C. รู้จัก Feedback Loops

**Reinforcing Loop (วงจรเสริม):**
```
ความยากจน → สุขภาพแย่ → ทำงานไม่ได้ → รายได้ลด → ความยากจน ↑
     ↑_________________________________________________↓
```

**Balancing Loop (วงจรต้านทาน):**
```
โรคระบาด ↑ → มาตรการควบคุม ↑ → โรคระบาด ↓
     ↑________________________________↓
```

### 3.3 ตัวอย่าง Systems Thinking ในสาธารณสุข

**ปัญหา: การขาดแคลนบุคลากรในพื้นที่ห่างไกล**

```
                    บุคลากรลาออก
                         ↓
    ความเหนื่อยล้า ← คนงานน้อย → คุณภาพบริการ ↓
         ↑                                    ↓
    สภาพแวดล้อม ↓                        ความพึงพอใจ ↓
    (ไฟฟ้า น้ำ ไอที)                          ↓
         ↑                          ไม่มีคนมาทำงาน
         └───────────────────────────────────┘
```

**จุดแทรกแซงที่เหมาะสม:**
1. ปรับปรุงสภาพแวดล้อมการทำงาน
2. สร้าง incentives ที่ดี
3. หมุนเวียนบุคลากร

## 4. กระบวนการทำ RCA

### 4.1 ขั้นตอน 6 ขั้น

```
┌────────────────────────────────────────────────────────────┐
│  1. DEFINE                                                  │
│     กำหนดปัญหาอย่างชัดเจน                                   │
├────────────────────────────────────────────────────────────┤
│  2. COLLECT                                                 │
│     รวบรวมข้อมูลและหลักฐาน                                  │
├────────────────────────────────────────────────────────────┤
│  3. ANALYZE                                                 │
│     วิเคราะห์สาเหตุด้วยเครื่องมือ RCA                        │
├────────────────────────────────────────────────────────────┤
│  4. VERIFY                                                  │
│     ยืนยันสาเหตุเชิงลึก                                     │
├────────────────────────────────────────────────────────────┤
│  5. DEVELOP                                                 │
│     พัฒนาแนวทางแก้ไข                                       │
├────────────────────────────────────────────────────────────┤
│  6. IMPLEMENT & MONITOR                                     │
│     นำไปปฏิบัติและติดตามผล                                  │
└────────────────────────────────────────────────────────────┘
```

### 4.2 รายละเอียดแต่ละขั้น

#### Step 1: Define the Problem

**เกณฑ์ SMART สำหรับการนิยามปัญหา:**

| ตัวอักษร | ความหมาย | ตัวอย่าง |
|---------|---------|---------|
| S | Specific | "อัตราตายมารดา" ไม่ใช่ "สุขภาพแย่" |
| M | Measurable | "80 ต่อ 100,000" |
| A | Actionable | สามารถแก้ไขได้ |
| R | Relevant | สำคัญต่อเป้าหมาย |
| T | Time-bound | "ปี 2566" |

**ตัวอย่างที่ดี:**
> "อัตราตายมารดาในจังหวัดเชียงราย 80 ต่อ 100,000 เกิดมีชีพ ในปี 2566 สูงกว่าค่าเฉลี่ยประเทศ 78%"

#### Step 2: Collect Data

**แหล่งข้อมูลที่ควรรวบรวม:**
- ข้อมูลเชิงปริมาณ (magnitude, trends)
- ข้อมูลเชิงคุณภาพ (interviews, observations)
- ข้อมูลบริบท (contextual factors)
- Best practices จากที่อื่น

#### Step 3: Analyze Causes

**เครื่องมือที่ใช้:** (จะอธิบายละเอียดใน Handout ถัดไป)
- 5 Whys
- Fishbone Diagram
- Social-Ecological Model
- Causal Loop Diagram

#### Step 4: Verify Root Causes

**เกณฑ์ของ Root Cause:**

| เกณฑ์ | คำอธิบาย |
|-------|---------|
| ความลึก | เป็นสาเหตุลึกสุดที่ควบคุมได้ |
| ความสามารถแก้ไข | ถ้าแก้ได้ ปัญหาจะไม่กลับมา |
| มีหลักฐาน | สนับสนุนด้วยข้อมูล |
| สอดคล้องกัน | ตรงกับ root causes อื่น |

#### Step 5: Develop Solutions

**SMART Solutions:**
- ระบุผู้รับผิดชอบ
- กำหนด Timeline
- ตั้งตัวชี้วัด
- ประเมินทรัพยากร

#### Step 6: Implement & Monitor

**ตัวชี้วัดการติดตาม:**
- Process indicators (กิจกรรมทำตามแผน)
- Outcome indicators (ผลลัพธ์เปลี่ยนไป)
- Impact indicators (ผลกระทบระยะยาว)

## 5. ข้อควรระวังในการทำ RCA

### 5.1 Pitfalls ที่พบบ่อย

| Pitfall | อาการ | แก้ไข |
|---------|-------|-------|
| Blame culture | โทษบุคคล | Focus ที่ระบบ |
| Analysis paralysis | วิเคราะห์มากเกิน | Set deadline |
| Confirmation bias | หาข้อมูลที่ยืนยันความเชื่อเดิม | ใช้หลายเครื่องมือ |
| Oversimplification | ลดความซับซ้อนมากไป | ยอมรับ complexity |
| Stopping too early | หยุดที่สาเหตุตื้น | ถาม "Why?" ต่อไป |

### 5.2 คำถามช่วยตรวจสอบ

- [ ] ถ้าแก้ root cause นี้ ปัญหาจะหายไปจริงหรือไม่?
- [ ] มี root cause ที่ลึกกว่านี้อีกหรือไม่?
- [ ] เรากำลังโทษบุคคลหรือดูที่ระบบ?
- [ ] มีหลักฐานสนับสนุนหรือเป็นการคาดเดา?

## แบบฝึกหัด: แยกแยะระดับของสาเหตุ

**ปัญหา:** อุบัติเหตุทางจราจรในพื้นที่ อปท. สูง

จัดประเภทสาเหตุต่อไปนี้:

| สาเหตุ | Immediate | Underlying | Root |
|--------|-----------|------------|------|
| คนขับเมาแล้วขับ | ☐ | ☐ | ☐ |
| ไม่มีระบบตรวจจับความเร็ว | ☐ | ☐ | ☐ |
| ถนนแคบไม่มีไฟฟ้าแสงสว่าง | ☐ | ☐ | ☐ |
| ขาดการบังคับใช้กฎหมาย | ☐ | ☐ | ☐ |
| วัฒนธรรมการดื่มสังสรรค์ | ☐ | ☐ | ☐ |
| ไม่มี public transport | ☐ | ☐ | ☐ |

**เฉลย:**
- Immediate: คนขับเมาแล้วขับ
- Underlying: ถนนแคบ, ไม่มีระบบตรวจจับ, ขาดการบังคับใช้กฎหมาย
- Root: วัฒนธรรมการดื่ม, ไม่มี public transport
