# Module 2.2: Causal Loop Diagram (CLD) - การวิเคราะห์ความสัมพันธ์เชิงระบบ
## เอกสารประกอบการสอน

---

## 1. ความเป็นมา

Causal Loop Diagram (CLD) เป็นเครื่องมือจาก Systems Thinking ที่ใช้แสดงความสัมพันธ์เชิงเหตุผลระหว่างตัวแปรต่างๆ ในระบบ ช่วยให้เห็นวงจรการเสริม (reinforcing) และการต้านทาน (balancing) ที่เกิดขึ้นในระบบสุขภาพ

---

## 2. สัญลักษณ์และความหมาย

### 2.1 องค์ประกอบพื้นฐาน

```
ตัวแปร A ──────────────────────────────► ตัวแปร B
          +
          (A ↑ → B ↑)

ตัวแปร A ──────────────────────────────► ตัวแปร B
          -
          (A ↑ → B ↓)
```

**สัญลักษณ์:**
- **ตัวแปร (Variable):** ปัจจัยที่มีผลต่อระบบ (วงกลมหรือสี่เหลี่ยม)
- **ลูกศร (Arrow):** แสดงทิศทางความสัมพันธ์
- **+ (Same):** ทิศทางเดียวกัน (A↑ → B↑ หรือ A↓ → B↓)
- **- (Opposite):** ทิศทางตรงกันข้าม (A↑ → B↓ หรือ A↓ → B↑)

### 2.2 ประเภทของ Loops

#### A. Reinforcing Loop (R) - วงจรเสริม
```
    ความยากจน
         +
         ↑
         │
    สุขภาพไม่ดี ←───+
         +          │
         ↓          │
    รายได้ลด ───────+
```

**ลักษณะ:** 
- ทบตัวเอง (self-reinforcing)
- ผลลัพธ์เพิ่มขึ้นเรื่อยๆ
- นับจำนวน "-" ใน loop: คู่ = Reinforcing

**ตัวอย่าง:**
- ความยากจน ↔ สุขภาพไม่ดี
- การติดเชื้อ ↔ ภูมิคุ้มกันต่ำ

#### B. Balancing Loop (B) - วงจรต้านทาน
```
    โรคระบาด
         +
         ↓
    มาตรการควบคุม
         +
         ↓
    โรคระบาดลด ←───+
         -          │
         ↑──────────┘
```

**ลักษณะ:**
- ควบคุม/stabilize ระบบ
- ทำให้เข้าสู่สมดุล
- นับจำนวน "-" ใน loop: คี่ = Balancing

**ตัวอย่าง:**
- โรคระบาด → การควบคุม → โรคลด
- ความต้องการ → การผลิต → ราคาลด

## 3. ขั้นตอนการสร้าง CLD

### Step 1: ระบุตัวแปรหลัก
เลือกตัวแปรที่เกี่ยวข้องกับปัญหา อย่างน้อย 3-5 ตัวแปร

### Step 2: กำหนดความสัมพันธ์
ระบุว่าตัวแปรใดมีผลต่อตัวแปรใด และทิศทางเป็นอย่างไร (+ หรือ -)

### Step 3: วาดโครงสร้าง
เชื่อมต่อตัวแปรด้วยลูกศรและสัญลักษณ์ + -

### Step 4: ระบุ Loops
หาว่ามีกี่ loop และเป็นแบบใด (R หรือ B)

### Step 5: วิเคราะห์พฤติกรรมระบบ
ทำนายว่าระบบจะมีพฤติกรรมอย่างไรในระยะยาว

## 4. ตัวอย่าง CLD ในสาธารณสุข

### 4.1 ตัวอย่างที่ 1: ความยากจนกับสุขภาพ

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   ความยากจน ────────────+───────────► สุขภาพไม่ดี            │
│       ▲                      +                              │
│       │                      │                              │
│       │                      ↓                              │
│       │                 ค่าใช้จ่ายสุขภาพ                     │
│       │                      +                              │
│       │                      │                              │
│       │                      ↓                              │
│       └────────────── รายได้ลด ◄────────── ทำงานไม่ได้      │
│                          +                                  │
│                                                             │
│   Loop: Reinforcing (R1) - "Poverty Trap"                   │
│   (มี - 0 ตัว = คู่ = Reinforcing)                          │
└─────────────────────────────────────────────────────────────┘
```

**การอ่าน:**
- ความยากจนเพิ่ม → สุขภาพไม่ดีเพิ่ม (+)
- สุขภาพไม่ดี → ค่าใช้จ่ายสุขภาพเพิ่ม (+)
- ค่าใช้จ่ายเพิ่ม → รายได้ลด (+) [เพราะใช้เงินไปกับรักษา]
- รายได้ลด → ทำงานไม่ได้ (+)
- ทำงานไม่ได้ → ความยากจนเพิ่ม (+)

**จุดแทรกแซง:**
1. ลดค่าใช้จ่ายสุขภาพ (UCS)
2. สร้างรายได้ระหว่างป่วย
3. ป้องกันโรคเพื่อลดความยากจน

### 4.2 ตัวอย่างที่ 2: การระบาดของโรค

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   จำนวนผู้ติดเชื้อ ────────+───────────► การแพร่เชื้อ         │
│        ▲                      +                             │
│        │                      │                             │
│        │                      ↓                             │

│        │                      +                             │
│        │                      │                             │
│        └──────────────────────┘                             │
│                                                             │

```

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   จำนวนผู้ติดเชื้อ ────────+───────────► มาตรการควบคุม       │
│        ▲                      +                             │
│        │                      │                             │
│        │                      ↓                             │
│        │                 จำนวนผู้ติดเชื้อลด                 │
│        │                      -                             │
│        │                      │                             │
│        └──────────────────────┘                             │
│                                                             │
│   Loop: Balancing (B1) - "Control Measures"                 │
│   (มี - 1 ตัว = คี่ = Balancing)                            │

```

### 4.3 ตัวอย่างที่ 3: การขาดแคลนบุคลากร

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   บุคลากรลาออก ◄─────── ความเหนื่อยล้า ◄────── คนงานน้อย   │
│       │                      ▲                      +       │
│       │                      │                      │       │

│       │                                                     │
│       ↓                                                     │
│   ความพึงพอใจลด ─────────────────────────► ไม่มีคนมาทำงาน   │
│       +                                    │                │
│       │                                    │                │
│       └────────────────────────────────────┘                │
│                                                             │

```

## 5. การใช้ CLD เพื่อหาจุดแทรกแซง

### 5.1 หลักการเลือกจุดแทรกแซง

```
1. หา link ที่มี influence สูง
2. หา link ที่เปลี่ยนแปลงได้ง่าย
3. หา link ที่มี leverage effect
4. พิจารณา delay ในระบบ
```

### 5.2 ตัวอย่างการเลือกจุดแทรกแซง

**ระบบ: การสูบบุหรี่ในวัยรุ่น**

```
การโฆษณาบุหรี่ ──+──► การสูบบุหรี่ ──+──► ความเสี่ยงโรค
      │                    │
      │                    +        (จุดแทรกแซงที่ 1)
      │                    ↓
      │              Peer influence
      │                    +
      │                    ↓
      └────────────── การสูบบุหรี่ (loop)
```

**จุดแทรกแซงที่มีประสิทธิผล:**
1. **ห้ามโฆษณาบุหรี่** - ตัดต้นตอ (high leverage)
2. **ลด Peer influence** - สร้างกลุ่มไม่สูบบุหรี่
3. **ให้ความรู้** - แต่มี leverage ต่ำกว่า

## 6. การใช้ CLD ร่วมกับเครื่องมืออื่น

### 6.1 CLD + Stock and Flow

```
CLD: แสดงความสัมพันธ์เชิงเหตุผล
    ↓
Stock and Flow: แสดงการสะสมและไหลของตัวแปร
    ↓
Simulation: ทำนายพฤติกรรมระบบในอนาคต
```

### 6.2 CLD + Root Cause Analysis

```
5 Whys: หา root cause ในแต่ละตัวแปร
    ↓
CLD: แสดงความสัมพันธ์ระหว่าง root causes
    ↓
หา common root cause ที่มี leverage สูง
```

## 7. ข้อควรระวัง

### 7.1 ข้อจำกัดของ CLD

| ข้อจำกัด | คำอธิบาย |
|---------|---------|
| ไม่แสดง magnitude | รู้ว่ามีผล แต่ไม่รู้ว่ามากน้อยแค่ไหน |
| ไม่แสดง delay | ผลลัพธ์อาจเกิดช้าหรือเร็ว |
| Simplified | ไม่ครอบคลุมทุกปัจจัย |
| Static | ไม่แสดงการเปลี่ยนแปลงตามเวลา |

### 7.2 วิธีแก้ไข

- ใช้ CLD ร่วมกับ quantitative data
- เพิ่ม time delay บน diagram
- ทำ sensitivity analysis
- ใช้ simulation software (Stella, Vensim)
