# Module 2.2: 5 Whys Technique - คู่มือการใช้งาน
## เอกสารประกอบการสอน

---

## 1. ความเป็นมา

**5 Whys** เป็นเทคนิคที่พัฒนาโดย Sakichi Toyoda ผู้ก่อตั้ง Toyota Industries ในปี 1930 เพื่อหาสาเหตุของปัญหาในกระบวนการผลิต ต่อมาถูกนำมาใช้อย่างแพร่หลายในด้านคุณภาพและสาธารณสุข

---

## 2. หลักการ

### 2.1 พื้นฐาน

ถามคำถาม "ทำไม?" (Why?) ซ้ำ 5 ครั้ง (หรือจนกว่าจะถึงสาเหตุเชิงลึก) เพื่อเจาะลึกจากอาการสู่สาเหตุที่แท้จริง

```
ปัญหา (Problem)
    ↓
ทำไม? → คำตอบ 1 (สาเหตุระดับ 1)
    ↓
ทำไม? → คำตอบ 2 (สาเหตุระดับ 2)
    ↓
ทำไม? → คำตอบ 3 (สาเหตุระดับ 3)
    ↓
ทำไม? → คำตอบ 4 (สาเหตุระดับ 4)
    ↓
ทำไม? → คำตอบ 5 (Root Cause)
```

### 2.2 ทำไมต้อง 5 ครั้ง?

จำนวน 5 ไม่ใช่กฎตายตัว แต่เป็นค่าเฉลี่ยที่พบว่าเพียงพอสำหรับการลงลึกถึงสาเหตุเชิงลึก อาจใช้ 3-7 ครั้ง ขึ้นอยู่กับความซับซ้อนของปัญหา

---

## 3. ขั้นตอนการใช้ 5 Whys

### Step 1: กำหนดปัญหา
เขียนปัญหาที่ต้องการวิเคราะห์ให้ชัดเจน เฉพาะเจาะจง

### Step 2: ถาม "ทำไม?" ครั้งแรก
ถามว่าทำไมปัญหานี้ถึงเกิดขึ้น

### Step 3: ตอบคำถาม
ให้คำตอบที่เป็นข้อเท็จจริง ไม่ใช่การคาดเดา

### Step 4: ถาม "ทำไม?" ต่อ
ถามต่อจากคำตอบที่ได้ จนกว่าจะถึงสาเหตุเชิงลึก

### Step 5: ตรวจสอบ Root Cause
ตรวจสอบว่าถ้าแก้ไขที่จุดนี้ ปัญหาจะหายไปจริงหรือไม่

## 4. ตัวอย่างการใช้ในสาธารณสุข

### ตัวอย่างที่ 1: อัตราตายมารดาสูง

```
ปัญหา: อัตราตายมารดาสูง 80 ต่อ 100,000 ในจังหวัด X

ทำไม? → มีภาวะแทรกซ้อนระหว่างตั้งครรภ์และคลอดมาก
    ↓
ทำไม? → มารดาไม่ได้รับการดูแลก่อนคลอดที่เหมาะสม
    ↓
ทำไม? → มารดาบางกลุ่มไม่เข้ารับบริการ ANC 5 ครั้ง
    ↓
ทำไม? → ไม่มีค่าเดินทางและขาดความรู้เรื่องสิทธิ UCS
    ↓
ทำไม? → ระบบข้อมูลไม่สามารถระบุกลุ่มเสี่ยงได้ล่วงหน้า

Root Cause: ขาดระบบ early warning สำหรับมารดากลุ่มเสี่ยง
```

**แนวทางแก้ไข:**
- พัฒนาระบบคัดกรองความเสี่ยง ANC
- สร้างระบบเตือนอัตโนมัติ
- จัดสรรค่าเดินทางสำหรับกลุ่มเสี่ยง

### ตัวอย่างที่ 2: โรคเบาหวานไม่ควบคุม

```
ปัญหา: 40% ของผู้ป่วยเบาหวานมี HbA1c > 9%

ทำไม? → ผู้ป่วยไม่ทานยาตามที่แพทย์สั่งอย่างสม่ำเสมอ
    ↓
ทำไม? → ผู้ป่วยไม่เข้าใจความสำคัญของการควบคุมน้ำตาล
    ↓
ทำไม? → ไม่ได้รับการให้คำแนะนำอย่างต่อเนื่อง
    ↓
ทำไม? → ระบบ follow-up ไม่มีประสิทธิภาพ
    ↓
ทำไม? → ไม่มีระบบ case management ที่เชื่อมโยง รพ.สต. กับ รพ.

Root Cause: ขาดระบบ care continuum สำหรับผู้ป่วย NCD
```

### ตัวอย่างที่ 3: การติดเชื้อในโรงพยาบาล

```
ปัญหา: อัตราติดเชื้อแผลผ่าตัดสูงกว่ามาตรฐาน

ทำไม? → พยาบาลลืมล้างมือก่อนทำแผล
    ↓
ทำไม? → ไม่มีเจลล้างมือใกล้ที่ทำแผล
    ↓
ทำไม? → งบประมาณซื้อเจลไม่เพียงพอ
    ↓
ทำไม? → ไม่มีการจัดสรรงบสำหรับ infection control
    ↓
ทำไม? → โรงพยาบาลไม่ให้ความสำคัญกับ patient safety

Root Cause: ขาดวัฒนธรรมความปลอดภัยของผู้ป่วย
```

## 5. เทคนิคการใช้ 5 Whys อย่างมีประสิทธิภาพ

### 5.1 DO's (ควรทำ)

| เทคนิค | คำอธิบาย |
|--------|---------|
| ถามคำถามที่ตรงประเด็น | เน้นที่ "facts" ไม่ใช่ "blame" |
| ใช้หลายคนช่วยกันคิด | Brainstorm เป็นทีม |
| เอกสารทุกขั้นตอน | บันทึกเพื่อตรวจสอบ |
| ตรวจสอบ root cause | ถ้าแก้แล้วปัญหาจะหายไปจริงไหม? |
| ใช้ evidence | อ้างอิงข้อมูลเมื่อเป็นไปได้ |

### 5.2 DON'Ts (ไม่ควรทำ)

| ข้อห้าม | ตัวอย่าง |
|---------|---------|
| โทษบุคคล | "เพราะพยาบาลขี้เกียจ" ❌ |
| หยุดเร็วเกินไป | หยุดที่สาเหตุตื้น |
| ถามแบบนำคำตอบ | "ทำไมพยาบาลไม่ล้างมือ?" (leading) |
| ตอบคลุมเครือ | ต้องเฉพาะเจาะจง |
| ละเลย context | ต้องพิจารณาบริบท |

### 5.3 การตรวจสอบคำตอบ

**คำถามสำหรับตรวจสอบ:**

1. **ถ้าแก้คำตอบนี้ ปัญหาจะหายไปหรือไม่?**
   - ถ้าไม่ → ถาม Why ต่อ
   - ถ้าใช่ → อาจเป็น root cause

2. **คำตอบนี้ควบคุมได้หรือไม่?**
   - ถ้าไม่ได้ → อาจต้องเปลี่ยนมุมมอง

3. **มี root cause อื่นที่สำคัญกว่าหรือไม่?**
   - ถ้ามี → ทำ 5 Whys หลายกิ่ง

## 6. การใช้ 5 Whys กับปัญหาที่มีหลายสาเหตุ

### 6.1 กรณีมีหลาย Root Cause

```
ปัญหา: เด็กอ้วนเพิ่มขึ้นในโรงเรียน

กิ่งที่ 1: ด้านโภชนาการ
ทำไม? → กินอาหารขยะบ่อย
ทำไม? → โรงเรียนขายอาหารที่ไม่มีประโยชน์
ทำไม? → ไม่มีนโยบายควบคุมอาหารในโรงเรียน
Root Cause: ขาดกฎหมายควบคุมอาหารโรงเรียน

กิ่งที่ 2: ด้านกายภาพ
ทำไม? → ไม่ออกกำลังกาย
ทำไม? → ไม่มีสนาม/เวลาออกกำลังกาย
ทำไม? → โรงเรียนเน้นวิชาการมากเกินไป
Root Cause: นโยบายการศึกษาไม่ส่งเสริมกิจกรรมกาย

กิ่งที่ 3: ด้านครอบครัว
ทำไม? → พ่อแม่ให้เงินซื้อขนมมาก
ทำไม? → พ่อแม่ทำงานไม่มีเวลาดูแล
ทำไม? → ไม่มีระบบสนับสนุนครอบครัว
Root Cause: ขาดนโยบาย work-life balance
```

### 6.2 การรวม Root Causes

หลังจากหา root causes ทุกกิ่งแล้ว ให้หา common root cause:

```
Root Causes:
1. ขาดกฎหมายควบคุมอาหารโรงเรียน
2. นโยบายการศึกษาไม่ส่งเสริมกิจกรรมกาย  
3. ขาดนโยบาย work-life balance

→ Common Root Cause: ไม่มี Health in All Policies (HiAP)
```

## 7. ตัวอย่างการประยุกต์ใช้

### 7.1 การใช้กับ Stakeholder ต่างระดับ

| ระดับ | ปัญหา | 5 Whys | Root Cause |
|-------|-------|--------|------------|
| ชาติ | ความเหลื่อมล้ำสุขภาพ | ... | การจัดสรรทรัพยากรไม่เป็นธรรม |
| จังหวัด | ขาดแคลนแพทย์ | ... | นโยบาย incentives ไม่ดี |
| อำเภอ | ส่งต่อผู้ป่วยล่าช้า | ... | ขาดระบบ communication |
| หมู่บ้าน | คนไม่มา รพ.สต. | ... | ไม่เชื่อมั่นบุคลากร |

### 7.2 การใช้กับประเด็นต่างๆ

| ประเด็น | ปัญหา | Root Cause ตัวอย่าง |
|--------|-------|-------------------|
| NCD | ผู้ป่วย DM ไม่มาติดตาม | ไม่มีระบบ appointment reminder |
| แม่และเด็ก | ทารกน้ำหนักต่ำ | ขาด nutrition counseling |
| อุบัติเหตุ | การบาดเจ็บจากการทำงาน | ไม่มี safety protocol |
| จิตเวช | ผู้ป่วยจิตเวชไม่กินยา | stigma ในชุมชน |

## แบบฝึกหัด

### แบบฝึกหัดที่ 1: ทำ 5 Whys

เลือกหนึ่งปัญหาจากรายการแล้วทำ 5 Whys:
- [ ] อัตราการสูบบุหรี่ในวัยรุ่นเพิ่มขึ้น
- [ ] ผู้ป่วยความดันโลหิตสูงไม่ควบคุม
- [ ] การตายจากโรคหัวใจในชนบทสูง
- [ ] การฆ่าตัวตายในวัยรุ่น

ตรวจสอบ 5 Whys ต่อไปนี้ว่าถูกต้องหรือไม่ ถ้าไม่ถูก ให้แก้ไข:

```

ทำไม? → คนกลัวผลข้างเคียง
ทำไม? → ได้ยินข่าวลือจากโซเชียลมีเดีย
ทำไม? → เพราะคนโง่
ทำไม? → เพราะการศึกษาต่ำ
ทำไม? → เพราะรัฐบาลไม่สนับสนุนการศึกษา

Root Cause: รัฐบาลไม่ทำงาน
```
