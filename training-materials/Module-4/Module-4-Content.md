## หัวข้อการสอน

### 4.1 โครงสร้างข้อเสนอแนะนโยบาย (3 ชั่วโมง)

**วัตถุประสงค์การเรียนรู้**
- เข้าใจโครงสร้างข้อเสนอแนะนโยบาย 7 ส่วน และเหตุผลที่ต้องมี
- เขียน Problem Statement พร้อมหลักฐานสนับสนุนและสรุปสาเหตุราก
- เชื่อม Recommendation กับ Theory of Change/Logic Model
- วางแผนนำไปใช้ พร้อม M&E และความเสี่ยง

**โครงสร้างเนื้อหา**
- ภาพรวมโครงสร้าง 7 ส่วน (Executive Summary ถึง M&E)
- Executive Summary ที่สั้นแต่ครบ
- Problem Analysis: Problem Statement + Evidence + Root Causes
- Stakeholder Analysis และ Policy Options (MCDA)
- Recommended Policy + SMART
- Theory of Change และ Logic Model
- Implementation Plan + Risk + M&E

**สาระสำคัญเชิงปฏิบัติ**
- ทุกส่วนต้องเชื่อมกันเป็นเรื่องเดียว: ปัญหา -> สาเหตุ -> ทางเลือก -> ข้อเสนอแนะ -> การนำไปใช้
- ใช้ข้อมูลแบบ baseline/target และระบุแหล่งที่มาให้ชัด
- ToC ตอบคำถาม "ทำไมถึงคิดว่าจะได้ผล" ส่วน Logic Model ตอบ "ทำอะไร-ได้อะไร"
- แผนการดำเนินงานต้องมี Timeline, งบประมาณ, ความเสี่ยง และตัวชี้วัด

### 4.2 การเขียนอย่างมีประสิทธิภาพ (3 ชั่วโมง)

**วัตถุประสงค์การเรียนรู้**
- เขียน Problem Statement และ Recommendation ให้ชัด กระชับ และน่าเชื่อถือ
- ใช้ข้อมูลสนับสนุนอย่างเหมาะสม พร้อมแสดงข้อจำกัด
- แปลงข้อเสนอแนะให้เป็น SMART Recommendation
- ปรับภาษาให้เหมาะกับผู้อ่านระดับผู้บริหาร

**โครงสร้างเนื้อหา**
- หลักการเขียนเชิงนโยบาย: ชัด-สั้น-ตรงประเด็น
- โครงสร้างการเล่าเรื่อง: ปัญหา -> หลักฐาน -> สาเหตุ -> ทางเลือก -> ข้อเสนอแนะ
- การใช้ข้อมูล: baseline/target, trend, benchmark
- การเขียน SMART Recommendations
- การจัดรูปแบบ (headings, bullets, visual cues)
- การจัดการข้อจำกัดของข้อมูล

**สาระสำคัญเชิงปฏิบัติ**
- “1 ประโยค = 1 ประเด็น” ลดศัพท์เทคนิค
- ข้อมูลต้องบอกที่มา และระบุปี
- ข้อเสนอแนะต้องระบุผู้รับผิดชอบ และกรอบเวลา
- เขียนแบบ “ผู้อ่านตัดสินใจได้ใน 2 นาที”

### 4.3 การนำเสนอและสื่อสารนโยบาย (2 ชั่วโมง)

**วัตถุประสงค์การเรียนรู้**
- ปรับสารตามผู้ฟังและช่องทางการสื่อสาร
- ใช้ storytelling และ data visualization เพื่อเพิ่มการยอมรับ
- เตรียมการตอบคำถามและจัดการข้อโต้แย้ง
- สร้างแรงสนับสนุนจากผู้มีส่วนได้ส่วนเสีย

**โครงสร้างเนื้อหา**
- Audience Mapping และ Message Map
- Storytelling Framework (Situation-Complication-Question-Answer)
- หลักการ Data Visualization
- โครงสร้างการนำเสนอเชิงนโยบาย
- การจัดการคำถามและการสร้างพันธมิตร

**สาระสำคัญเชิงปฏิบัติ**
- “หนึ่งสไลด์ = หนึ่งประเด็น”
- ตัวเลขสำคัญไม่เกิน 2-3 ตัวต่อสไลด์
- ระบุ Next Steps และผู้รับผิดชอบให้ชัด
- เตรียมคำตอบสำหรับคำถามยาก 5 ข้อ
