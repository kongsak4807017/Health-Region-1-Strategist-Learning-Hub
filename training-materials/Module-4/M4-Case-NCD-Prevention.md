# Case Study: ข้อเสนอแนะนโยบายป้องกันโรคไม่ติดต่อ (NCD)
## กรณีศึกษา: โรคเบาหวานในจังหวัดเชียงใหม่

---

## บทนำ

### บริบท
โรคไม่ติดต่อเรื้อรัง (NCDs) โดยเฉพาะโรคเบาหวาน เป็นปัญหาสาธารณสุขสำคัญของประเทศไทย ในจังหวัดเชียงใหม่ พบว่ามีผู้ป่วยเบาหวานจำนวนมากและมีแนวโน้มเพิ่มขึ้นอย่างต่อเนื่อง ควบคุมไม่ได้ตามเป้าหมาย

### ข้อมูลทั่วไป
- **พื้นที่:** จังหวัดเชียงใหม่ (ประชากร ~1.78 ล้านคน)
- **ปัญหา:** โรคเบาหวานไม่ควบคุม
- **ปีที่ศึกษา:** 2566

---

## ส่วนที่ 1: Problem Analysis

### 1.1 Problem Statement

ในขณะนี้ ผู้ป่วยเบาหวานในจังหวัดเชียงใหม่ (ประมาณ 120,000 คน) มีอัตราการควบคุมเบาหวานต่ำเพียง 35% (HbA1c < 7%) ซึ่งต่ำกว่าเป้าหมายของกระทรวงสาธารณสุข (50%) ส่งผลให้มีภาวะแทรกซ้อนเพิ่มขึ้น ค่าใช้จ่ายด้านสุขภาพสูงถึง 1,200 ล้านบาทต่อปี เนื่องจากระบบการติดตามติดตามผู้ป่วยไม่มีประสิทธิภาพ ผู้ป่วยขาดความรู้ในการจัดการตนเอง และการเข้าถึงบริการในพื้นที่ห่างไกลจำกัด

### 1.2 ข้อมูลสนับสนุน

| ตัวชี้วัด | ค่าที่พบ | เป้าหมาย | แหล่งข้อมูล |
|----------|---------|---------|------------|
| จำนวนผู้ป่วย DM | 120,000 คน | - | NHSO 2566 |
| Control rate (HbA1c < 7%) | 35% | 50% | MOPH |
| HbA1c เฉลี่ย | 8.5% | < 7% | รพ. |
| ภาวะแทรกซ้อนไตวาย | 15% | < 10% | รพ. |
| ค่าใช้จ่าย/คน/ปี | 10,000 บาท | - | NHSO |

### 1.3 Root Cause Analysis

**5 Whys Analysis:**

```
ปัญหา: ผู้ป่วย DM ควบคุมไม่ได้ (HbA1c สูง)

ทำไม? → ไม่ทานยาตามใบสั่ง และไม่ปรับพฤติกรรม
    ↓
ทำไม? → ไม่เข้าใจโรค และไม่เห็นความสำคัญ
    ↓
ทำไม? → ไม่ได้รับคำแนะนำอย่างต่อเนื่อง
    ↓
ทำไม? → ระบบ follow-up ไม่มีประสิทธิภาพ
    ↓
ทำไม? → ไม่มีระบบ case management ที่เชื่อมโยง รพ.สต. กับ รพ.

Root Cause: ขาดระบบ care continuum สำหรับผู้ป่วย NCD
```

**Fishbone Diagram:**

```
                         [DM ไม่ควบคุม]
                              |
    +------------+------------+------------+
    |            |            |            |
  People      Process       Policy      Place

People:
- ผู้ป่วย: ไม่เข้าใจโรค ไม่ทานยา ไม่ออกกำลังกาย
- ผู้ดูแล: ไม่มีความรู้
- บุคลากร: เวลาไม่พอ ไม่ได้รับการอบรม

Process:
- วินิจฉัย: ตรวจไม่สม่ำเสมอ
- รักษา: ปรับยาไม่ตามผล
- Follow-up: ไม่มีระบบนัด/เตือน

Policy:
- ยา: บัญชีจำกัด ไม่ครอบคลุมยาใหม่
- เครื่องมือ: HbA1c ไม่ครอบคลุมทุกครั้ง
- บุคลากร: จำนวนไม่พอ

Place:
- รพ.สต.: ไม่มีเครื่องตรวจน้ำตาล
- ที่บ้าน: ไม่มีเครื่องวัด ไม่มีตาชั่ง
- ชุมชน: อาหารไม่เหมาะสม
```

## ส่วนที่ 3: Policy Options Analysis

### 3.1 ทางเลือกที่พิจารณา

**Option A: เพิ่มบุคลากรใน รพ.**
- รายละเอียด: จ้างแพทย์ผู้เชี่ยวชาญโรคเบาหวานเพิ่ม
- ข้อดี: มีผู้เชี่ยวชาญดูแล
- ข้อเสีย: ต้นทุนสูง แพทย์ขาดแคลน
- ต้นทุน: 50 ล้านบาท/ปี

**Option B: ขยายบัญชียา**
- รายละเอียด: เพิ่มยาใหม่ใน NHSO list
- ข้อดี: ผู้ป่วยได้ยาดีขึ้น
- ข้อเสีย: ต้นทุนงบประมาณสูงมาก
- ต้นทุน: 200 ล้านบาท/ปี

**Option C: ระบบ Case Management (ทางเลือกที่แนะนำ)**
- รายละเอียด: พยาบาลดูแลผู้ป่วยรายบุคคล เชื่อมโยง รพ.สต.-รพ.
- ข้อดี: ตรงจุด ต้นทุนต่ำ ผลยั่งยืน
- ข้อเสีย: ต้องอบรมบุคลากร
- ต้นทุน: 20 ล้านบาท/ปี

## ส่วนที่ 4: Recommended Policy

### 4.1 ข้อเสนอแนะที่เลือก

**โครงการ: "พยาบาลเบาหวานรายบุคคล" (Diabetes Case Management)**

**เป้าหมาย:**
- เพิ่ม control rate จาก 35% เป็น 60% ภายใน 3 ปี
- ลด HbA1c เฉลี่ยจาก 8.5% เหลือ 7.5%
- ลดภาวะแทรกซ้อนไตวายร้อยละ 30

### 4.2 กลยุทธ์หลัก

| กลยุทธ์ | กิจกรรม | ผลลัพธ์ |
|--------|--------|---------|
| 1. Risk Stratification | แบ่งกลุ่มผู้ป่วย ติดตามตามความเสี่ยง | ผู้ป่วยเสี่ยงสูงได้รับการดูแลใกล้ชิด |
| 2. Self-Management Support | อบรมผู้ป่วยและครอบครัว | ผู้ป่วยสามารถจัดการตนเองได้ |
| 3. Care Coordination | พยาบาล case manager เชื่อมโยง รพ.สต.-รพ. | ไม่มีช่องโหว่ในการดูแล |
| 4. Technology Support | แอปติดตามน้ำตาล เตือนการนัด | ผู้ป่วยติดตามตนเองได้ |

### 4.3 Theory of Change

```
กิจกรรม → ผลผลิต → ผลลัพธ์ → ผลกระทบ

อบรมผู้ป่วย → ความรู้เพิ่ม → พฤติกรรมดีขึ้น → HbA1c ลด
Case Manager → ติดตามครบ → ไม่ขาดนัด → ควบคุมได้
Technology → เข้าถึงข้อมูล → ตัดสินใจดีขึ้น → ภาวะแทรกซ้อนลด

Assumptions:
1. ผู้ป่วยยอมเปลี่ยนพฤติกรรม
2. บุคลากรผ่านการอบรมแล้ว
3. เทคโนโลยีใช้งานได้จริง
```

## บทเรียนที่ได้

### ปัจจัยความสำเร็จ
1. การมีระบบ case management ที่ชัดเจน
2. การใช้เทคโนโลยีช่วยติดตาม
