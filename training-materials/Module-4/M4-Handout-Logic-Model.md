## เอกสารประกอบการสอน

## 1. ความแตกต่างระหว่าง Logic Model กับ Theory of Change

| | Theory of Change | Logic Model |
|---|-----------------|-------------|
| **โฟกัส** | เหตุผลเชิงทฤษฎีว่าทำไมถึงเกิดผล | กระบวนการทำงานจากอินพุตสู่ผลลัพธ์ |
| **ลักษณะ** | ครอบคลุม ซับซ้อน มีบริบท | เป็นระเบียบ เป็นเส้นตรง |
| **การใช้** | วางแผนกลยุทธ์ อธิบายสาเหตุ | ติดตามผล รายงาน |
| **สมมติฐาน** | เน้นมาก | เน้นน้อย |

**ความสัมพันธ์:** Theory of Change เป็น "หัวใจ" ส่วน Logic Model เป็น "โครงสร้าง"

## 2. Theory of Change (ToC)

### 2.1 ความหมาย

Theory of Change คือ กรอบแนวคิดที่อธิบายว่า **ทำไม** และ **อย่างไร** ที่การดำเนินการนำไปสู่การเปลี่ยนแปลงที่ต้องการ โดยแสดง:
- ความเชื่อ/สมมติฐาน (Assumptions)
- เงื่อนไขที่จำเป็น (Preconditions)
- เส้นทางสู่การเปลี่ยนแปลง (Pathway of change)

### 2.2 องค์ประกอบหลัก

```
┌─────────────────────────────────────────────────────────────────┐
│                    THEORY OF CHANGE                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  CONTEXT (บริบท)                                                │
│  • สถานการณ์ปัจจุบัน                                           │
│  • ปัจจัยแวดล้อม                                               │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  LONG-TERM GOAL (เป้าหมายระยะยาว)                               │
│  • ผลกระทบที่ต้องการ (Impact)                                  │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  PATHWAY OF CHANGE (เส้นทางสู่การเปลี่ยนแปลง)                  │
│                                                                 │
│  สมมติฐาน → เงื่อนไข 1 → เงื่อนไข 2 → ... → เป้าหมาย          │
│     ↑          ↑           ↑                                    │
│     └──────────┴───────────┘                                    │
│         INTERVENTIONS (กิจกรรม)                                 │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ASSUMPTIONS (สมมติฐาน)                                         │
│  • สิ่งที่เชื่อว่าเป็นจริง                                      │
│  • ปัจจัยที่ต้องมีเพื่อให้สำเร็จ                               │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  INDICATORS (ตัวชี้วัด)                                         │
│  • วัดความก้าวหน้า                                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**โครงการ: ลดอัตราตายมารดา**

├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  OUTCOMES (ผลลัพธ์)                                             │
│  ระดับชุมชน: มารดาเข้าถึงบริการ ANC ทันเวลา                    │
│       ↓                                                         │
│  ระดับบริการ: ภาวะแทรกซ้อนจัดการได้รวดเร็ว                     │
│       ↓                                                         │
│  ระดับระบบ: การส่งต่อฉุกเฉินมีประสิทธิภาพ                      │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  INTERVENTIONS (กิจกรรม)                                        │
│  • คัดกรองกลุ่มเสี่ยง                                          │
│  • ติดตามอัตโนมัติ                                             │
│  • ส่งต่อฉุกเฉิน                                               │

│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ASSUMPTIONS (สมมติฐาน)                                         │
│  1. มารดายอมรับการเข้าร่วมโครงการ                              │
│  2. บุคลากรมีความสามารถจัดการภาวะฉุกเฉิน                       │
│  3. ระบบขนส่งพร้อมใช้งาน                                        │
│  4. งบประมาณไม่ถูกตัด                                          │
│                                                                 │

### 2.4 การเขียน Theory of Change

**ขั้นตอน:**

1. **กำหนดเป้าหมายระยะยาว:** ต้องการเห็นอะไรเปลี่ยนแปลง?
2. **ย้อนกลับจากเป้าหมาย:** ต้องมีอะไรก่อนหน้าเป้าหมาย?
3. **ระบุเงื่อนไข (Preconditions):** อะไรต้องเกิดขึ้นก่อน?
4. **กำหนดกิจกรรม:** ทำอะไรเพื่อให้เงื่อนไขเกิดขึ้น?
5. **ระบุสมมติฐาน:** อะไรที่ต้องเชื่อว่าเป็นจริง?
6. **ตั้งตัวชี้วัด:** วัดความก้าวหน้าอย่างไร?

## 3. Logic Model

### 3.1 ความหมาย

Logic Model คือ แผนภาพที่แสดงความสัมพันธ์ระหว่างทรัพยากรที่ลงทุน กิจกรรมที่ทำ และผลที่ได้รับ

### 3.2 องค์ประกอบ

```
INPUTS → ACTIVITIES → OUTPUTS → OUTCOMES → IMPACT
(ทรัพยากร)  (กิจกรรม)   (ผลผลิต)   (ผลลัพธ์)   (ผลกระทบ)
   ↓          ↓          ↓          ↓           ↓
เงิน       อบรม      จำนวนที่    พฤติกรรม   การเปลี่ยน
บุคลากร   ให้บริการ   อบรม       เปลี่ยน    แปลงระดับ
อุปกรณ์   รณรงค์     จำนวนที่     ความรู้    ประชากร
สถานที่   ติดตาม     ให้บริการ   เพิ่มขึ้น   สุขภาพดี
```

### 3.3 อธิบายแต่ละส่วน

| ส่วน | คำถาม | ตัวอย่าง |
|-----|-------|---------|
| **Inputs** | มีอะไร? | งบ 10 ล้านบาท บุคลากร 20 คน |
| **Activities** | ทำอะไร? | อบรม คัดกรอง ติดตาม |
| **Outputs** | ได้อะไร? | อบรม 500 คน คัดกรอง 1,000 คน |
| **Outcomes** | เปลี่ยนอะไร? | ความรู้เพิ่ม พฤติกรรมดีขึ้น |
| **Impact** | ผลสุดท้าย? | อัตราตายลด สุขภาพดีขึ้น |

### 3.4 ตัวอย่าง Logic Model

**โครงการ: ควบคุมเบาหวานในชุมชน**

```
┌─────────────────────────────────────────────────────────────────┐
│                        LOGIC MODEL                              │
├──────────┬───────────┬────────────┬──────────────┬──────────────┤
│ INPUTS   │ ACTIVITIES│ OUTPUTS    │ OUTCOMES     │ IMPACT       │
├──────────┼───────────┼────────────┼──────────────┼──────────────┤
│ • งบ 5 ล้าน│ 1. อบรม   │ 1. อบรมผู้ │ Short-term:  │ ลดภาวแทรก  │
│ • พยาบาล │    อสม.   │    ป่วย   │ • ความรู้    │ กซ้อนจาก    │
│   5 คน  │    5 ครั้ง│    200 คน  │   เพิ่ม 80%  │ เบาหวาน    │
│ • เครื่อง│ 2. คัด    │ 2. คัด     │ • สามารถ    │ ลดการนอน   │
│   วัดน้ำ │    กรอง   │    กรอง   │   ตรวจน้ำตาล│ โรงพยาบาล  │

│   รพ.สต.│    กลุ่ม  │    10 กลุ่ม│   ลดลง      │             │
│         │    ผู้    │    4 ครั้ง/│ • ไป รพ.     │             │
│         │    ป่วย   │    ปี      │   น้อยลง    │             │
│         │ 4. ติด    │ 4. ติดตาม  │ Long-term:   │             │
│         │    ตาม    │    ครบ    │ • ภาวะแทรก  │             │
│         │    ราย    │    80%     │   กซ้อนลด   │             │
│         │    บุคคล  │            │              │             │
└──────────┴───────────┴────────────┴──────────────┴──────────────┘
```

### 3.5 ความสัมพันธ์ระหว่างระดับ

```
Inputs (เราควบคุมได้) 
    ↓
Activities (เราทำ)
    ↓
Outputs (เราผลิต)
    ↓
Outcomes (ผู้รับบริการเปลี่ยน)
    ↓
Impact (สังคมเปลี่ยน - ใช้เวลานาน)
```

**ความสามารถในการควบคุม:**
- Inputs-Outputs: ควบคุมได้โดยตรง
- Outcomes: อิทธิพลได้มาก
- Impact: อิทธิพลได้บางส่วน มีปัจจัยอื่นร่วม

## 4. การสร้าง Logic Model

### 4.1 ขั้นตอน

```
ขั้นที่ 1: กำหนดปัญหาและเป้าหมาย
              ↓
ขั้นที่ 2: ทำงานย้อนหลังจาก Impact → Inputs
              ↓
ขั้นที่ 3: กรอกแต่ละช่อง
              ↓
ขั้นที่ 4: ตรวจสอบความสมเหตุสมผล
              ↓
ขั้นที่ 5: ระบุสมมติฐานและปัจจัยภายนอก
              ↓
ขั้นที่ 6: กำหนดตัวชี้วัดแต่ละระดับ
```

### 4.2 ตัวชี้วัด (Indicators)

**คุณสมบัติตัวชี้วัดที่ดี (SMART):**

| ระดับ | ตัวชี้วัดตัวอย่าง | SMART |
|------|------------------|-------|
| Output | จำนวนผู้ได้รับอบรม | อบรม 200 คน ภายใน 6 เดือน |
| Outcome | % ที่มีความรู้เพิ่ม | เพิ่ม 80% ภายใน 1 ปี |
| Impact | ลดอัตราตาย | ลด 20% ใน 3 ปี |

### 4.3 สมมติฐาน (Assumptions)

**ตัวอย่างสมมติฐานในโครงการสุขภาพ:**

1. **ผู้รับบริการยอมรับ:**
   - มารดายอมมา ANC
   - ผู้ป่วยยอมเปลี่ยนพฤติกรรม

2. **บุคลากรมีความสามารถ:**
   - ผ่านการอบรมแล้ว
   - ไม่ลาออกก่อน

3. **ระบบสนับสนุน:**
   - งบประมาณไม่ถูกตัด
   - อุปกรณ์ไม่เสีย

4. **บริบทเอื้อ:**
   - ไม่มีโรคระบาดใหญ่
   - การเมืองไม่เปลี่ยน

## 5. การใช้ Logic Model ในการทำงาน

### 5.1 การวางแผน

- ตรวจสอบว่าทรัพยากรเพียงพอหรือไม่
- ตรวจสอบความสมเหตุสมผลระหว่างกิจกรรมและผล
- ระบุช่องโหว่และความเสี่ยง

### 5.2 การติดตาม

- ติดตาม Output: ทำตามแผนหรือไม่?
- ติดตาม Outcome: เปลี่ยนแปลงหรือไม่?
- ปรับแผนตามผลที่ติดตามได้

### 5.3 การประเมิน

- ประเมินประสิทธิผล (Effectiveness)
- ประเมินประสิทธิภาพ (Efficiency)
- เรียนรู้บทเรียน

## แบบฝึกหัด

### แบบฝึกหัดที่ 1: สร้าง Theory of Change

เลือกหนึ่งโครงการแล้วสร้าง Theory of Change:
- [ ] โครงการลดการสูบบุหรี่ในวัยรุ่น
- [ ] โครงการเพิ่มการฝากครรภ์

### แบบฝึกหัดที่ 2: สร้าง Logic Model

จาก Theory of Change ที่สร้าง:
1. แปลงเป็น Logic Model
