# Module 4.1: โครงสร้างข้อเสนอแนะนโยบาย (Policy Recommendation Structure)
## เอกสารประกอบการสอน - ฉบับละเอียด

---

## 1. ความสำคัญของโครงสร้างที่ดี

ข้อเสนอแนะนโยบายที่ดีต้องมีโครงสร้างชัดเจน เพื่อให้ผู้อ่านสามารถ:
- เข้าใจปัญหาและบริบทได้รวดเร็ว
- เห็นความสมเหตุสมผลของข้อเสนอแนะ
- ตัดสินใจได้อย่างมีข้อมูลครบถ้วน
- นำไปสู่การปฏิบัติได้จริง

---

## 2. โครงสร้าง 7 ส่วนหลัก

```
┌─────────────────────────────────────────────────────────────────┐
│  1. EXECUTIVE SUMMARY                                           │
│     (บทสรุปผู้บริหาร - 1 หน้า)                                  │
├─────────────────────────────────────────────────────────────────┤
│  2. PROBLEM ANALYSIS                                            │
│     (การวิเคราะห์ปัญหา)                                         │
├─────────────────────────────────────────────────────────────────┤
│  3. STAKEHOLDER ANALYSIS                                        │
│     (การวิเคราะห์ผู้มีส่วนได้ส่วนเสีย)                          │
├─────────────────────────────────────────────────────────────────┤
│  4. POLICY OPTIONS ANALYSIS                                     │
│     (การวิเคราะห์ทางเลือกนโยบาย)                                │
├─────────────────────────────────────────────────────────────────┤
│  5. RECOMMENDED POLICY                                          │
│     (ข้อเสนอแนะที่เลือก)                                        │
├─────────────────────────────────────────────────────────────────┤
│  6. IMPLEMENTATION PLAN                                         │
│     (แผนการนำไปปฏิบัติ)                                         │
├─────────────────────────────────────────────────────────────────┤
│  7. MONITORING & EVALUATION                                     │
│     (การติดตามและประเมินผล)                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 3. รายละเอียดแต่ละส่วน

### 3.1 Executive Summary (บทสรุปผู้บริหาร)

**ความยาว:** 1 หน้า A4 (ไม่เกิน 2 หน้า)

**ส่วนประกอบ:**

| หัวข้อ | รายละเอียด | ความยาว |
|--------|-----------|---------|
| **ปัญหา** | อธิบายปัญหาหลัก พร้อมตัวเลขสำคัญ | 2-3 ประโยค |
| **สาเหตุหลัก** | ระบุ 2-3 สาเหตุเชิงลึก | 2-3 ประโยค |
| **ข้อเสนอแนะ** | สรุปทางเลือกที่แนะนำ | 2-3 ประโยค |
| **ประโยชน์** | ผลลัพธ์ที่คาดว่าจะได้รับ | 2-3 ประโยค |
| **ทรัพยากร** | งบประมาณ/เวลาคร่าวๆ | 1-2 ประโยค |

**ตัวอย่างที่ดี:**

> **ปัญหา:** อัตราตายมารดาในจังหวัด X สูงถึง 80 ต่อ 100,000 เกิดมีชีพ (สูงกว่าค่าเฉลี่ยประเทศ 78%) โดยเฉพาะในกลุ่มหญิงตั้งครรภ์ที่ไม่ได้รับ ANC ครบ 5 ครั้ง
>
> **สาเหตุหลัก:** (1) ขาดระบบ early warning สำหรับมารดากลุ่มเสี่ยง (2) ไม่มีกลไกช่วยเหลือค่าเดินทาง (3) ระบบ referral ฉุกเฉินไม่มีประสิทธิภาพ
>
> **ข้อเสนอแนะ:** ดำเนินโครงการ "พยาบาลครรภ์อันตรายรายบุคคล" ด้วย 4 กลยุทธ์หลัก: (1) คัดกรองกลุ่มเสี่ยง (2) ติดตามอัตโนมัติ (3) ส่งต่อฉุกเฉิน (4) ชดเชยค่าเดินทาง
>
> **ประโยชน์:** ลดอัตราตายมารดาเหลือต่ำกว่า 50 ต่อ 100,000 ภายใน 3 ปี พร้อมเพิ่ม ANC coverage จาก 60% เป็น 90%
>
> **ทรัพยากร:** งบประมาณ 10 ล้านบาทต่อปี ระยะเวลา 3 ปี

### 3.2 Problem Analysis (การวิเคราะห์ปัญหา)

#### A. Problem Statement

**รูปแบบการเขียน:**
```
ในขณะนี้ [กลุ่มเป้าหมาย] กำลังประสบกับ [ปัญหา] 
ซึ่งมี [ผลกระทบ] เนื่องจาก [สาเหตุเบื้องต้น]
```

**ตัวอย่าง:**
> ในขณะนี้ หญิงตั้งครรภ์ในจังหวัด X (ประชากร 500,000 คน) กำลังประสบกับอัตราการตายระหว่างตั้งครรภ์และคลอดที่สูงถึง 80 ต่อ 100,000 เกิดมีชีพ (ค่าเป้าหมาย SDG: 70) ซึ่งสูงกว่าค่าเฉลี่ยประเทศ (45) เกือบ 2 เท่า เนื่องจากการเข้าถึงบริการ ANC และการคลอดไม่เพียงพอ

#### B. หลักฐานข้อมูล (Evidence)

**ตารางสรุปข้อมูล:**

| ตัวชี้วัด | ค่าที่พบ | ค่าเป้าหมาย | แหล่งข้อมูล | ปี |
|----------|---------|------------|------------|-----|
| MMR | 80/100,000 | <50 | MOPH | 2566 |
| ANC 5 visits | 60% | 90% | NHSO | 2566 |
| Delivery in facility | 70% | 95% | MOPH | 2566 |

#### C. Root Cause Analysis

**นำเสนอผลการวิเคราะห์ RCA:**
- 5 Whys
- Fishbone Diagram (อาจแนบเป็นภาคผนวก)
- สรุป Root Causes 3-5 ข้อ

### 3.4 Policy Options Analysis (การวิเคราะห์ทางเลือกนโยบาย)

#### A. ทางเลือกที่พิจารณา

**ตัวอย่าง:**

**Option A: สร้างโรงพยาบาลศูนย์เพิ่ม**
- รายละเอียด: สร้าง รพ.ศูนย์ 1 แห่งในพื้นที่ห่างไกล
- ข้อดี: มีศักยภาพสูง รองรับภาวะซับซ้อน
- ข้อเสีย: ใช้งบสูง (500 ล้านบาท) ใช้เวลานาน (5 ปี)
- ต้นทุน: 500 ล้านบาท + ดำเนินการ 100 ล้าน/ปี

**Option B: แกรนท์ชุมชน**
- รายละเอียด: ให้ทุนอำเภอจัดการบริการเอง
- ข้อดี: ยืดหยุ่น ตอบสนองพื้นที่
- ข้อเสีย: ความไม่เท่าเทียม ขาดมาตรฐาน
- ต้นทุน: 20 ล้าน/อำเภอ/ปี

**Option C: High-risk tracking (ทางเลือกที่แนะนำ)**
- รายละเอียด: ระบบติดตามมารดากลุ่มเสี่ยงรายบุคคล
- ข้อดี: ตรงจุด ต้นทุนต่ำ ผลเร็ว
- ข้อเสีย: ต้องมีระบบข้อมูลที่ดี
- ต้นทุน: 10 ล้าน/ปี

#### B. ตารางเปรียบเทียบ MCDA

| Effectiveness (30%) | 4 | 3 | 5 |

| Sustainability (10%) | 4 | 3 | 4 |

### 3.5 Recommended Policy (ข้อเสนอแนะที่เลือก)

#### A. สาระสำคัญ

**ชื่อโครงการ:** "พยาบาลครรภ์อันตรายรายบุคคล"

**เป้าหมาย:** 
- ลดอัตราตายมารดาเหลือ <50/100,000 ภายใน 3 ปี
- เพิ่ม ANC 5 visits จาก 60% เป็น 90%

**กลุ่มเป้าหมาย:** หญิงตั้งครรภ์กลุ่มเสี่ยงในจังหวัด X

#### B. กลยุทธ์หลัก

| กลยุทธ์ | กิจกรรม | ผู้รับผิดชอบ | ตัวชี้วัด |
|--------|--------|------------|----------|
| 1. คัดกรอง | ประเมินความเสี่ยง ANC ทุกครั้ง | รพ.สต. | % คัดกรองครบ |
| 2. ติดตาม | ระบบเตือนอัตโนมัติ | สปสช. | % กลุ่มเสี่ยงได้ติดตาม |
| 3. ส่งต่อ | ทีมสูติอุบัติเหตุ 24 ชม. | รพ.ทั่วไป | เวลาส่งต่อ |
| 4. ชดเชย | ค่าเดินทางกลุ่มเสี่ยง | อปท. | จำนวนผู้ได้รับ |

#### C. Theory of Change

```
Inputs (ทรัพยากร) → Activities (กิจกรรม) → Outputs (ผลผลิต) → Outcomes (ผลลัพธ์) → Impact (ผลกระทบ)

งบ 10 ล้านบาท    คัดกรอง          มารดากลุ่มเสี่ยง      ANC เพิ่ม          MMR ลด
บุคลากร         ติดตาม           ได้รับการดูแล      ส่งต่อเร็วขึ้น      ความ equity ดีขึ้น
ระบบ IT        ส่งต่อ           ครบถ้วน           ภาวะแทรกซ้อนลด
              ชดเชย
```
