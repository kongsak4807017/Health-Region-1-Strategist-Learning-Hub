## โครงสร้างข้อเสนอแนะที่ครบถ้วน

```
1. Executive Summary
   - สรุปปัญหาและข้อเสนอแนะหลัก
   - ผลประโยชน์ที่คาดว่าจะได้รับ
   - ต้นทุน/งบประมาณคร่าวๆ

2. Problem Analysis
   - Problem Statement ที่ชัดเจน
   - หลักฐานเชิงข้อมูล
   - Root Causes

3. Stakeholder Analysis
   - ผู้มีส่วนได้ส่วนเสีย
   - ตำแหน่งและอิทธิพล

4. Policy Options Analysis
   - ทางเลือกที่พิจารณา
   - การวิเคราะห์ MCDA

5. Recommended Policy
   - ข้อเสนอแนะที่เลือก
   - Theory of Change
   - Logic Model

6. Implementation Plan
   - Timeline
   - งบประมาณ
   - ความเสี่ยงและแผนรองรับ

7. Monitoring & Evaluation
   - KPIs
   - LogFrame
   - แหล่งข้อมูล
```

## หลักการ SMART สำหรับ Recommendations

| ตัวอักษร | ความหมาย | ตัวอย่างที่ดี | ตัวอย่างที่ไม่ดี |
|----------|---------|-------------|---------------|
| **S**pecific | เฉพาะเจาะจง | "พัฒนาระบบ case management สำหรับผู้ป่วย DM" | "พัฒนาระบบสุขภาพ" |
| **M**easurable | วัดได้ | "เพิ่ม control rate จาก 35% เป็น 60%" | "เพิ่ม control rate" |
| **A**chievable | เป็นไปได้ | "ภายใน 3 ปี" | "ภายใน 3 เดือน" |
| **R**elevant | สัมพันธ์กับปัญหา | "ลดภาวะแทรกซ้อนไตวาย" | "เพิ่มความพึงพอใจพนักงาน" |
| **T**ime-bound | มีกรอบเวลา | "ต.ค. 2568" | "เร็วๆ นี้" |

## Theory of Change Framework

### ขั้นตอน

1. **ระบุ Long-term Goal**
   - ผลกระทบที่ต้องการเห็นในระยะยาว

2. **วิเคราะห์ Pre-conditions**
   - อะไรต้องเกิดขึ้นก่อนจึงจะบรรลุเป้าหมาย

3. **ออกแบบ Intervention**
   - กิจกรรมที่จะทำให้เกิด pre-conditions

4. **ระบุ Assumptions**
   - สมมติฐานที่ต้องเป็นจริง

### ตัวอย่าง

```
[กิจกรรม] → [ผลผลิต] → [ผลลัพธ์] → [ผลกระทบ]

Case Management → ติดตามครบ → Control rate 60% → ภาวะแทรกซ้อนลด

Assumptions:
- ผู้ป่วยยอมเปลี่ยนพฤติกรรม
- บุคลากรผ่านการอบรม
```

## Logic Model Framework

### Components

| Component | คำถาม | ตัวอย่าง |
|-----------|------|---------|
| Inputs | อะไรที่ใส่เข้าไป? | งบ 46 ล้าน, พยาบาล 50 คน |
| Activities | ทำอะไร? | อบรม, case management |
| Outputs | ผลผลิตอะไร? | ผู้ป่วย 10,000 คนได้รับบริการ |
| Outcomes | เปลี่ยนแปลงอะไร? | Control rate 60% |
| Impact | ผลกระทบระยะยาว? | ภาวะแทรกซ้อนลด 30% |

## การเขียน Policy Brief

### โครงสร้าง (2-4 หน้า)

1. **Title** - ชัดเจน ดึงดูด
2. **Executive Summary** - ครึ่งหน้า
3. **Context** - บริบทและความสำคัญ
4. **Problem** - Problem Statement
5. **Options** - ทางเลือกที่พิจารณา
6. **Recommendations** - ข้อเสนอแนะหลัก
7. **Implementation** - การดำเนินงานคร่าวๆ
8. **Contact** - ผู้ติดต่อ

### Dos & Don'ts

**✓ ควรทำ:**
- ใช้ภาษาง่าย
- มีข้อมูลสนับสนุน
- ใช้ bullet points
- มี call-to-action

**✗ ไม่ควรทำ:**
- ใช้ศัพท์เทคนิคมาก
- ยาวเกิน 4 หน้า
- ไม่มีตัวเลข
- ไม่ระบุผู้รับผิดชอบ
